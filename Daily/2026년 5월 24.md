대주제: C#에서 값을 저장하고 구분하는 기본 방법
소주제 1: 변수란 무엇인가?
 변수란, 프로그램이 동작할 때 필요한 데이터를 저장하는 일종의 임시 저장 공간이다.
변수를 만들 때, 즉 처음 선언할 때에는 반드시 이 변수의 이름과 이 변수가 담을 데이터의 형식을 지정해야 한다.
위의 정의만으로는 감이 잘 안 잡힐 수 있으니 조금 더 구체적인 상황을 가정하면, 내가 RPG 게임을 만들고 있고, 기초적인 칼을 만든다고 해보자.
이때, 칼에는 여러 가지 동작 기능들이 있겠지만 오직 공격과 관련된 데이터에 한해서만 생각해본다면 칼의 공격력, 칼의 사거리, 공격 쿨타임 등이 있을 수 있다. 
위의 3가지 데이터를 변수를  통해서 메모리에 임시로 저장하고 관리한다고 가정한다고 했을 때, 아래처럼 변수를 선언할 수 있다.
flaot attackCooltime = 0.3f;
flaot attackPoint = 320f;
float attackDistance = 32f;

 위의 상황에서처럼 변수를 만들 땐 이름과 데이터 형식을 명시해야 한다. 이름은 우리가 마음대로 짓는다고하지만, 그렇다면 데이터 형식을 우리가 마음대로 만들어서 지정이 가능한가??
음.. 가능하다. 

소주제 2: 변수를 선언할 때, 데이터 형식을 정하는 기준은 무엇인가?
 데이터 형식에 대해 알아야 우리가 마음편히 지정이 가능할 것이다. 
우선, 데이터 형식은 크게 2가지이다. 
 첫 번째는 C#에서 제공해주는 기본 형식인 primitive 형식이다.
우리가 흔히 쓰는 변수인 int, float, char 등이 여기에 속한다. 
 
 두 번째는 사용자가 직접 지정하는 사용자 지정 형식이다. 
지금은 그냥 "아, 데이터 형식도 만들 수 있구나라고 이해하고 넘어가자."

소주제 3: 변수 그냥 선언하기만 되는가?
 결론적으로만 보면, 아니다. 변수를 선언하면 반드시 값을 할당하는 초기화라는 과정을 거쳐야 한다. 
변수라는 것도 결국에는 메모리를 차지하는 녀석인데 이놈이 일도 안 하고 자리만 차지한다고 생각해보라. 가득이나 바쁜 Ram의 입장에서는 죽여버리고 싶을 것이다.
그래서 C#에서는 이런 상황을 방지하기 위해서 초기화 시키지 않는 데이터는 아얘 에러로 띄운다. 그리고 프로그램이 돌아가면서 더 이상 안 쓰이는 데이터는 가비지 콜렉터라는 일종의 메모리 청소부가 버려버린다.

소주제 4: 변수에 저장되는 값은 뭐라고 하는가?
 리터럴이라고 한다. 변수의 데이터 형식은 리터럴의 형식에 종속적인 관계다. 
근데, 아무런 값 자체를 가지지 않는 null도 있다.

소주제 5: 초기화는 반드시 선언시 진행되야 하는가?
 아니다. 일반적으로는 많이 그렇게 되기도 하지만, 값이 유돌적으로 바뀌거나 하는 복잡한 상황에서는 처음에는 선언만 하고, 나중에 초기화가 이루어진다. 

소주제 6: 데이터 형식의 층위
 정수와 실수, 문자와 문자열 같이 같은 카테고리 안에 있는 데이터 형식들은 에로사항이 있다. 작은 층위에 속한 형식의 데이터는 큰 형식의 데이터 형식으로 바꿀 수 있지만, 그 반대는 안된다. 

 소주제 7: 변수 선언은 노가다인가?
  상황에 따라 다르다. 
  int a,b,c; 도 문법 상 가능하고, 
  int a;
  int b;
  int c;
  역시 가능하다. 스타일과 상황에 맞추어서 적절히 사용하면 된다.

  그리고 아래처럼 초기화하는 것도 가능하다. 
  int a,b,c;
  a=b=c=10;

소주제 8: 변수는 알겠는데, 상수는 뭔가?
 상수란, 우리가 알고 있는 수학의 그 의미와 비슷하다. 상수는 한 번 선언되면 바꿀 수 없다. 그래서 선언과 동시에 초기화 되어져야 한다.
변수 앞에 const 기호를 붙히면 상수를 선언할 수 있다. 

소주제 9: 같은 카테고리의 형식인데 왜 종류가 여러 개인가?
 같은 카테고리더라도, 부호와 장 용량에 따라 형식이 여러 개다. 다 외울 필요는 없고, 리터럴의 특성을 고려해서 잘 정하면 된다. 
실전에서는 어차피 int, float가 많이 쓰인다. 그때 그때 찾아 보는게 좋다.
